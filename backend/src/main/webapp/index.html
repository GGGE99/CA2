Request Body and Response Formats

User format (don’t provide ID, for POST { "id": Number, "birthday": Date, "name": String, "gender": String [“male” | “Female”], "email": String (email), "phone": Phone, "address": Address, "hobby": Hobby }

Hobby format(don’t provide id for POST) { "id": Number, "name": String, "wikiLink": String, "type": String }

zip format { "zip": String, "city": String }

Errors

(e) All errors are reported using this format (with the HTTP-status code matching the number)

{ status : statusCode, "msg": "Explains the problem" }
(e1) :{ status : 404, "msg": "No content found for this request" }
(e2) :{ status : 400, "msg": "Field ‘xxx’ is required" } (for example, no name provided)
